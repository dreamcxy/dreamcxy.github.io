C++ 传递方式对比

Posted on 2020-07-02

C++ 几种传递方式探讨

传递方式

  • 值传递
  • 引用传递
  • 指针传递
  • 全局变量传递（其实是指两个函数可以访问到这个变量就行，比如类中的成员变量传递，或者使用static关键字去定义传递，或者用namepace去做，虽叫全局实非全局，这个效率在四者之间的效率是为最高）

比较

功能上

  • 值传递只是传递实参的拷贝，实参本身不改变

传递效率（被调用函数代码将实参传递到被调函数体内的过程）

  • 这个其实跟传递的对象尺寸有关，指针传递反正就是指针的大小，32位CPU 4字节， 64位CPU 8字节，而值的大小完全看参数了。引用传递最显然的实现方式是使用指针做的，但有些情况下编译器可以直接做到直接寻址的方式来做，这种情况下，效率跟上面的全局变量传递最快。
  • 如果是自定义类型的话，传值的话，生成临时对象的时候会执行构造函数，销毁时又要执行析构函数，如果成员比较多，构造和析构做的事有比较多，效率上大打折扣，远不如其他两种传递方式。当然，这种情况下，引用传递也是有可能做到直接寻址的，所以最优的还是引用传递，毕竟它最差也就是指针传递的效率了。

执行效率（被调用函数体内执行的效率）

  • 值传递复制到被调用函数体内之后，生成临时对象之后，函数流程都是直接的直接寻址做的，所以最快，指针和大多数情况下的引用都是走的间接寻址的方式。所以如果函数体内对变量操作频繁，大多数的引用和指针都会有明显的的效率丢失。

类型安全

  • 值传递和引用传递的过程中都会有强类型检查，所以安全，但指针传递类型检查较弱，如果是这种

    test(void *) 的，那么只要是指针就是合法的。

  • 但在多态的情况下，如果形参是父类，实参是子类，值传递会只构造父类的部分。

参数检查

  • 对内建的数据类型走值传递，自定义的请使用引用传递

灵活性

  • 指针最灵活

共有技术

const

  • 不允许修改

默认值

  • 我觉得特别好用，真的，特别是你接手前人的代码，需要扩展某个函数的时候，这时候用默认值，可以帮你省去大量的重写，不知道为什么C#会去掉这个特性
